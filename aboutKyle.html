  • Mangl3r (C++)
    Mangl3r is a multi-effect, multiband distortion plugin. Each band has 5 different distortion modules. They is a Saturator, a bitcrusher, a waveshaper, a clipper, and a wavefolder. The waveshaper has four different algorithms, the clipper has six, and the wavefolder has two, for a total of 14 different algorithms the user can choose from. The modules are draggable, which allows the user to rearrange how the input signal gets processed on the fly. Each module has it's own dry/wet mix, where 100% is only the distorted signal, and 0% is the original signal. Each module can also have it's signal boosted before and after the module processes the input. Above, a frequency analyzer can be seen to get a rough idea of what your sound is doing. To the left, the two knobs control the width of the three bands. Allowing for boarder or more precise tuning. Each band can be bypassed, meaning all distortion effects are turned off, or muted, which complexly removes the signal in that band. Finally, the RMS values are displayed middle right. The user can also change the input signal before any processing, or after all processing, as well as the overall mix of the plugin. The user can also change the plugin to be single band, which process the audio as a whole.

    Source code can be found here https://github.com/KiTiKDub/Mangl3r

  • Synth (C++)
    My 'Simple' Synth. This synth is equipped with two oscillators, which can each be made into sine, saw, triangle, and square waves. An FM oscillator modulates the 1st oscillator, that can also be made into the 4 waves of oscillator 1 and 2. There are three filter options, all that are run in series with each other. There is a ladder filter, which incorporates, low, high, and band passes. A phaser filter, and a comb filter. Below those are two LFO's, that can modulate most everything on the synth. The rate can be from .1 to 10 hz, and have the same four types of waves. The bottom lines of knobs change when the filter type is selected to match. Finally, at the top are global controls. These turn the oscillators on an off, the filters on and off, as well as the global gain.

    Source code can be found here https://github.com/KiTiKDub/SimpleSynth

  • 3-band Compressor (C++)
    Multiband compressor with bands for low, mids, and highs. Each band has adjustable crossover frequencies to dial in exactly where you need your compression.

    Each band has adjustable attack, release, threshold, and ratio, with the ability to mute, solo, and bypass the band.

    Visualize your compression with a FFT spectrum analyzer to see where you may need to dial down your sound. Each band has the threshold mapped to the analyzer so you can see exactly when your compression will start. The analyzer will also show the response of the compression so you can see exactly how it is responding to your input.

    Source code can be found here https://github.com/KiTiKDub/MBComp
  • Distortion (C++)
    Simple tanh distortion algorithm. The Drive knob controls the gain multiplier pushed through the tanh function. The range is a multiplier to the drive. e.g, .5 Range with a full Drive will be half the drive. Blend is a Wet/Dry knob. If the blend is at .5, an equal amount of distortion and clean signal will be put through. All the way up, only the distorted signal will go through. And volume is the overall gain of the plugin.

    Source code can be found here https://github.com/KiTiKDub/SimpleDistortion
  • Delay (C++)
    Simple Delay Algorithm. The top two knobs control the time of the delay. The larger the number, the larger the data that is stored and then delayed. The button in the middle can be clicked to allow for different delay lengths in the left and right channels. The feedback increases the amound of the delayed signal that is feedback into the delay. There are two dry wet algorithms. One proportionally decreases the dry signal as the wet signal increases. The other increases the wet signal while the dry signal stays intact.

    Source code can be found here https://github.com/KiTiKDub/SimpleDelay
  • Reverb (C++)
    Simple reverb algorithm. The size knob creates the size of the room, which results in how the reflections behave with the reverb. The damping knob controls the time that the reflections last. The width changes how wide the sound is, with 0% making the reverb mono. And the dry/wet allows for the change in how much of the signal is reverb. The Logo can be clicked to freeze the current reverb sound

    Source code can be found here https://github.com/KiTiKDub/SimpleReverb
